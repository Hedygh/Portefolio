# Sprint Planning and Project Management

**Project:** OG Game Challenge

**Project Duration:** June 15, 2026 – June 30, 2026

---

# Project Management Approach

The project followed a lightweight Agile workflow divided into two main development sprints.

Development tasks were distributed according to each team member's area of expertise, allowing parallel progress while reducing development conflicts.

Progress was monitored continuously through GitHub commits, regular communication between team members, and integration testing.

# Sprint 1 Priorities (MoSCoW)

## Must Have

- User authentication
- REST API
- Front-end interface
- Playable JavaScript games
- Database integration

## Should Have

- Leaderboard
- Responsive interface
- API documentation

## Could Have

- UI improvements
- Additional visual effects
- Code refactoring

## Won't Have

- Multiplayer
- Cloud deployment
- Additional game modes

# Task Dependencies

| Task | Depends On |
|------|------------|
| Frontend authentication | Backend API |
| Score submission | Backend endpoints |
| Leaderboard | Database + API |
| Game integration | Frontend interface |
| Final MVP | Successful integration of all components |

---

# Sprint Monitoring

Progress was monitored through:

- GitHub commits
- Feature branches
- Pull request reviews
- Regular team communication
- Continuous integration testing

---

# Sprint Review

At the end of each sprint, completed features were reviewed before moving to the next phase.

The final review confirmed:

- Backend APIs operational
- Front-end connected to the backend
- JavaScript games integrated
- Authentication functional
- Leaderboard operational
- MVP ready for demonstration
