## 2. Brainstorming and Idea Evaluation

### Brainstormed Ideas

During the ideation phase, the team explored several project concepts before selecting the final MVP.

#### 1. OG Game Challenge *Selected*

A competitive web platform where users can create a player profile, play mini-games, save their scores, and compete on a live leaderboard. The platform is designed to offer short but challenging skill-based games with increasing difficulty.

#### 2. Interactive AI Playground

A guided web platform where users can build simple AI-based interactions such as mini chatbots through step-by-step modules. The application would provide an interactive coding interface and live preview system.

#### 3. Tourist Review Platform

A web platform where users can browse famous tourist destinations around the world and post structured reviews and comments. The application would act as a community-based tourism feedback platform.

### Why Some Ideas Were Rejected

#### Interactive AI Playground
Although innovative and technically interesting, this idea was considered too complex for the project timeline. Building interactive AI tools without strong prior knowledge in machine learning or advanced AI systems would introduce significant technical risk and increase delivery uncertainty.

#### Tourist Review Platform
This idea was feasible and easier to implement, but the team found it less engaging and less aligned with their interests. While practical, it was considered too conventional and lacked the originality and excitement the team wanted for this project.

### Why OG Game Challenge Was Selected

OG Game Challenge was selected because it offered the best balance between feasibility, creativity, technical challenge, and team motivation.

It was chosen because:

* it is strongly aligned with the team’s shared interest in gaming
* it combines backend, frontend, API design, and interactive JavaScript logic
* it is technically ambitious while remaining realistic for an MVP
* it is more engaging and original than the alternative ideas
* it allows the team to build a fun product while demonstrating strong fullstack skills

## 3. Decision and Refinement

### Problem Statement

Many casual users enjoy short competitive experiences but lack simple web-based platforms that combine accessibility, challenge, and persistent progression. Most small browser games offer entertainment, but few provide structured competition, score persistence, and player identity.

### Proposed Solution

OG Game Challenge is a web-based gaming platform where users can create a player profile, play mini-games, save their performance, and compete through a dynamic leaderboard system.

The platform combines accessibility, competition, and progression in a lightweight and engaging browser experience.

### Target Audience

The target audience is broad and includes:

* casual players
* students
* competitive users looking for quick challenges
* users seeking short and accessible browser-based games

The platform is designed for anyone looking for a simple but competitive gaming experience.

### Application Type

OG Game Challenge is a **responsive web application** composed of:

* a dynamic frontend interface
* a backend API
* a persistent database
* browser-based JavaScript mini-games

The application is designed for desktop-first usage and may later be adapted for mobile responsiveness.

### Key MVP Features

The MVP will focus on three core features:

1. **User Registration and Player Profile**
   Users can create an account using email and define a player identity.

2. **Mini-Games Integration**
   Users can access and play multiple browser-based mini-games with increasing difficulty.

3. **Dynamic Leaderboard System**
   Player scores are saved and ranked in real time through a persistent leaderboard.

### SMART Objectives

* Build a functional user registration and authentication system for player accounts
* Develop and integrate 1 to 3 playable JavaScript mini-games
* Store and update player scores in a persistent database
* Display a live leaderboard ranking users by score
* Deliver a functional MVP with a complete gameplay loop by the end of the project timeline

### Project Scope

#### In Scope

The MVP will include:

* user registration and login
* player profile creation
* score persistence
* leaderboard system
* 1 to 3 mini-games
* responsive web interface
* backend API and database integration

#### Out of Scope

The MVP will not include:

* live multiplayer gameplay
* in-game chat system
* mobile application
* advanced social features
* advanced avatar customization
* AI opponents
* microtransactions or monetization systems

### Risks and Mitigation

| Risk                                                 | Impact                                | Mitigation                                                               |
| ---------------------------------------------------- | ------------------------------------- | ------------------------------------------------------------------------ |
| JavaScript mini-games become too complex or unstable | Delays and gameplay bugs              | Start with simple game mechanics and iterate progressively               |
| Poor database structure affects score reliability    | Data inconsistency and ranking issues | Design database schema early and validate score logic before integration |
| Limited development time                             | Incomplete MVP delivery               | Keep scope narrow and prioritize core features only                      |
